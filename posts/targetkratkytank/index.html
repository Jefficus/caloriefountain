    Target "Kratky Tank"

    Jefferson Smith

    2023-03-03 18:40

    GOAL: Build a Kratky-style growing tank with approx 4 sqft of crop space, and environment monitoring gear.

    Purpose: to be used for crop verification experiments.

    Sensors:

    • temperature
    • humidity
    • water depth
    • water conductivity
    • light intensity

    Deliverables:

    • tank, opaque
    • lid, opaque
    • internal root crop support rack
    • LED grow light, with control API client
    • sensor pack, operational
    • controller module, operational
      • receives/logs sensor data fm multiple tanks
      • controls lights on multiple tanks
    • 3 days worth of baseline data tracking

    Alterations (2023-03-18)

    The original plan for Kratky tanks was to build all my monitors from sensor chips, so I'd be able to replicate a consistent instrument network throughout my lab inexpensively, but this would leave me trusting home-built equipment with no way to verify them.

    Consequently, the Kratky Tank target will use off-the-shelf sensor gear.

    • InkBird ITDS-01 that will measure TDS, EC, and water temperature
    • HM Digital PH-200 to measure pH (and also temperature).

    Data Collection

    Since I'm not deploying automated sensors yet, the recording module can be manual too. So long as I have a system that lets me collect the manual entries easily and stores them in a way that will be data-compatible with later automated systems, I should be fine.

    To that end, I've built a data collection interface in Memento DB that I can run on my phone. This makes it very quick to capture the data, which means I will use it more frequently than I would a more cumbersome process.

    Lights

    For lights, I picked up a Roleadro 75W LED grow light but I quickly grew worried that 75W wouldn't be enough, so I replaced it with a FECiDA 1000W Dimmable LED system

    My initial target was to create containers of appx 4 sqft, but I've since realized that these are problematic for some experiments, because they allow root networks to intermingle. I'll certainly be testing these form factors later, but for my initial tests, I need things to be more independent, so I'll start with one container per plant.

    I‚Äôd like to build my own containers for testing, using my 3D printer, so I can adapt and evolve the design as my needs (and experience) evolve. I know this is doable, but my initial test was a failure. I printed a 50mm cube canister in PLA, with standard print settings, filled it with water, and set it on a piece of tissue (Kleenex), thinking that if it leaked at all, the tissue would reveal the leak.

    Well, it has been 4 days now and I just discovered that all the water is gone, but there was never any moisture visible on the container walls or distortions of the tissue. Upon closer inspection, however, the tissue has stiffened and developed a subtle watermark texture that wasn't evident to my earlier, more cursory inspections. It seems it didn't so much leak as osmose straight through the walls of the container.

    I'm going to try again, but this time I'll be using PETG filament, and changing the print settings to hopefully eliminate the osmotic pathways.

    Trial Tagging

    To make it easier to track trials, collect data, and avoid the predictable confusion of having 20 identical test containers, I'm using tracking tags with barcode identifiers. These barcodes are scannable by my data entry tool (described above) which can then quickly pre-load the entry screen with the pertinent unit id and be ready for my readings.

    For the record (in case anybody else is using Linux and wants to know about my toolchain) my tracking codes are simply the datestamp (to the second and without punctuation) of when the sample tag was printed. For example, the tag shown in the picture here is 20230318170214. This is printed over USB to a Dymo LabelManager 280, using the opensource dymoprint python script available here, and rendered in the code128 barcode format.

    (For completeness, the command to produce the barcodes is: dymoprint -c code128 "$(date +%Y%m%d%H%M%S)")
